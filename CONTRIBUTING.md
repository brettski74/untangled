# Contributing

The project is set up with rules, skill and workflows designed to work within Cursor. We use Cursor to help refine requirements and implement code, test and deployment automation. If you plan to work on the project as part of the development team, then expect to need a Cursor subscription and to follow the workflows provided in the project to do your work.

If you are submitting a pull request from your own fork, there's no strict requirement to use AI to develop your patch, but if you plan to develop your patches manually or by some other means, familiarize yourself with the rules, standards, architecture and guidelines encompassed in (AGENTS.md)[AGENTS.md], the (Cursor rules)[.cursor/] and other documentation in the project. Your patch stands a better chance of being accepted if you do.
